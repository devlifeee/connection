# Статус реализации по ТЗ

Обновлено: текущая ветка разработки. Ниже — что готово, что ещё предстоит, и как проверять.

## Что реализовано
- Обнаружение узлов: mDNS, список пиров через `/presence/peers`, запасной путь — `/peers`.
- Чат:
  - Прямые сообщения по libp2p, ACK, сохранение истории, read markers.
  - Мультихоп через relay (pubsub), отображение «через: N» в UI.
  - Outbox с ретраями и TTL, метрики (/metrics).
- Файлы:
  - Offer/Accept/Chunk/Ack/Complete, SHA‑256, возобновление, персист состояния.
  - Лимиты скорости: глобальные и per‑peer.
  - Trust‑only toggle для исходящих и фильтр входящих.
- Звонки (WebRTC):
  - Сигналинг поверх libp2p, авто‑accept (опционально), обработка ICE.
  - Метрики звонков и простая оценка качества (good/medium/poor).
- Безопасность:
  - Noise шифрование транспорта, блок‑лист.
  - Trust‑лист и режим «только доверенные» (файлы/медиа).
- UI/Dev:
  - Скрипт демо `npm run multi` поднимает 3 узла и 2 фронта.
  - Линт и билд фронта проходят; Go‑тесты — зелёные.

## Недоделано/доделать
- Методика проверок качества звонков: вынести в отдельный document c сценариями (пакетные потери, высокая задержка).
- Индикатор качества звонка в UI — расширенный (RTT, jitter, loss, битрейт с таймсериями).
- Опциональный «relay‑only» режим для демонстрации маршрутизации.

## Как проверить
1. Демонстрация
   - `npm run multi` — поднимет 3 ноды и 2 UI (порты 5183/5184).
   - В UI: список контактов, чат, файлы, звонки.
2. Чат
   - Отправить сообщение узлу без прямой связи — должно прийти через relay (бейдж «через: N»).
   - Проверить read‑маркеры (открыть чат на втором UI).
3. Файлы
   - Отправить файл, перезапустить агент — возобновление продолжается.
   - Включить trust‑only: недоверенным исходящие блокируются; входящие с недоверенных — отклоняются.
4. Звонки
   - Исходящий audio/video; проверить, что приходят события `incoming_call` и `hangup`.
   - Оценка качества в UI — отображается статус good/medium/poor.
5. Метрики
   - `/metrics` на HTTP‑порту агента: рост счётчиков outbox, calls.

## Известные фиксированные баги
- Фантомные ноды в списке: добавлена фильтрация устаревших записей и дедуп по PeerID на `/presence/peers`.
- media/call мог падать без JSON‑ошибки при отсутствии менеджера — добавлен безопасный обработчик и понятная ошибка.

## Где искать проблемы
- Сеть/топология: `node-agent/internal/runtime`, `internal/discovery`, `internal/api/server.go (/peers, /presence)`.
- Чат/relay: `internal/protocol`, `internal/chatrelay`, UI компоненты `ChatPanel`.
- Файлы: `internal/filetransfer`, соответствующие API‑ручки в `server.go`.
- Звонки: `internal/media`, обработчики в `server.go`, UI `CallsPanel`.

## Команды
- Веб:
  - `npm run lint`, `npm run build`, `npm run dev`.
- Бэкенд:
  - `cd node-agent && go build ./... && go test ./...`.

## Замечания
- Режим trust‑only по умолчанию выключен (для стабильных тестов). Включается через `/config/trust_only` или в UI в настройках.
